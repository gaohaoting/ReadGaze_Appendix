# Maintainer: anonymity before public GitHub / 4open

Before pushing to a **public** repository or registering **anonymous.4open.science**, verify:

- No author names, affiliations, or grant numbers in any file.
- No email addresses, institution domains, or personal homepages.
- Repository name and GitHub username should not identify the lab (use a neutral repo name).
- **Git history**: for a clean anonymized history, consider squashing or using a fresh repo with a single initial commit containing only scrubbed files.
- LICENSE file: if you include one, avoid putting identifiable legal names in the copyright line until de-anonymization after acceptance; use a generic SPDX identifier or omit until camera-ready, per your counsel.

This appendix package was authored to avoid PII by default; session IDs in reports are study codes, not participant names.
